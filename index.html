Curling Team Websites

TeamInglis.ca TeamBrunton.ca

Purpose

These web applications were created to serve as a globally available, and accessible platforms to access everything that has to do with a specific curling team. Prior to the development of these sites, information about curling teams is only primarily available through Facebook, X (formerly Twitter), or Instagram profiles, which limited viewership to those that had accounts on the appropriate platform. These applications also open the door to experiment and develop novel solutions to enhance the viewing experience for the amateur-level of play. Notably, it provides a way to seamlessly watch a live stream (from YouTube or Facebook), while seeing the line scores on the same screen. This functionality was previously only available when a game has been professionally streamed, or if it was streamed through a third-party service, such as OBS, and constantly monitored by a moderator. Additionally, when these sites are paired with my custom-developed auto-scoring system that pulls scores from various publicly available live scoring providers, website managers are able to "set it and forget it" and allow the scores to update themselves as the game progresses.

Execution

This website was created using the React framework as a foundation, with Material UI elements for the front-end. Google Firestore and Analytics are used throughout the application to provide real-time data to the user and analytics information for performance monitoring. The theming that is used throughout the application was made to coincide with the overall branding of the team that has already been established, however, it was implemented in such a way as to be easily modifiable if the need arises.

Inspiration

The inspiration of this application came from my first-hand knowledge of the inadequacies with the current landscape of curling websites and platforms, and the possibilities to grow the viewer base. The intention with this narrow-scope application is to expand the functionalities to a point that numerous teams can utilize the platform and grow the audience of the sport.

Wilderness Survival App

Purpose

This project was developed as part of my fourth-year capstone project, with a group of 3 individuals. The application was designed and intended to improve vital tracking for wilderness first responders. The current method for recording and tracking vitals in an emergency situation in the wilderness is a pen and paper (a standardized form, if available). This method has obvious inherent flaws that can make the process of tracking vitals a hindrance to the care that needs to be provided to the wounded and sick.

Execution

The application was built using React Native for a seamless development process for both Android and iOS platforms. MongoDB / MongoDB Realm was the selected choice as a database and authentication provider. The application provides tools to aid with vital tracking such as: a standardized workflow to guide the user through the basic set of vitals, custom reminders to retake a vital and, timers to assist in recording time-based vitals such as pulse or respiration. In addition, the app processes data for a patient into interactive graphs that can help visualize trends in a patient's vitals as well as identify changes that may not be obvious to the responder. Static resources are also available in the application to help a user prepare for a wilderness adventure with items such as: checklists to ensure all appropriate equipment is packed, how-to guides to ensure care taking procedures are up-to-date, and a small wilderness survival encyclopedia to be used as reference when providing care. Patient data is also able to be exported to a PDF to be shared with secondary responders such as paramedics or medical transport personnel. The application and all of it's functionalities are available to the user "out-of-the-box", however, if the user wishes to save their data for use across multiple devices, they are able to setup an account at any time.

Inspiration

The inspiration for the development of this application came from the identification of the flaws and "old-school" practices that are in place for wilderness survival. While these practices are tried and tested, they can always be improved, and in some cases, are in desperate need of a refinement.


Trusty Product Reviews

Purpose

This web application was created with a team of 5 students for a course on project management. The application allows users to review products (star rating), follow other users, and filter reviews based on other reviewers that are like-minded (Jaccard distance) or users that they follow.

Execution

This application was developed as a Single Page Application (SPA) with the Java programming language, utilizing the Spring framework with an integrated h3 database. The application was compiled and deployed using Maven and Heroku (since the changes to Heroku's terms of service (Nov 2022), the deployment is no longer visible, as it was deployed on a free tier).

Inspiration

The inspiration for this project came from the want / need to see reviews that are relevant to the user, instead of the reviews that are relevant only to the product. This application, allows for the "old way" of skimming through reviews, as well as allowing users to see reviews from users that are likely to be interested in the same products, and have the same tastes as you. This allows for a more biased rating that will divert the user way from products that are more-likely to disappoint them in-particular, and attract them toward product that they will find interesting.


LookInnaBook

Purpose

This application was created with a team of 2 members for a final project in a database management course during my undergrad. The intention was to have an application that allows users to manage and peruse a library of books. The application also required that users with admin privileges are able to modify and update the stock and availability of books as necessary.

Execution

The application was developed largely using the Java Swing library for front-end (GUI) development, and a PostgreSQL database as the data storage solution. It featured screens for customers to peruse the available books, add them to a cart, complete billing and shipping information, and "complete" an order (this was not actually functional because that wasn't the point, but the foundation is there). For the administrators, the application has the functionality to modify all aspects of the books, view purchasing reports, order more books, and fulfil customer orders.
